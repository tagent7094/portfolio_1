MODULAR SCORING OVERRIDES — ATTACH TO ANY OPENING EVALUATOR
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
These rules hard-enforce calibration. Apply deterministically before final score output.

1. CALIBRATION ANCHORS
- 5 = average LinkedIn hook. Most score here.
- 7 = you stop scrolling and read the next line.
- 8 = you stop + feel something + screenshot it.
- 9-10 = exceptionally rare. Requires exact quote, specific moment, or undeniable tension.

2. HARD CAPS
- Abstract framing ("Many founders struggle with...") → cap at 5
- Platform clichés ("Hot take", "Unpopular opinion") → cap at 4
- Body diverges from opening promise within 3 lines → body_fit = 4, score -= 1
- Credential establishment opener ("I build X", "I've done Y") when source did NOT open with credentials → cap at 5, humanness cap at 5

3. AI-SLOP DETECTION LIST
"Here's the thing" / "Here's what people miss" / "Let that sink in" / "What if I told you" / "In today's landscape" / "Game changer" / "The truth is" / "Real talk" / "Hard truth" / starts with emoji / >1 exclamation mark / rhetorical question opener / "Buckle up" / "Let me tell you" / "Nobody talks about"
If ANY match: slop_detected = true, score -= 2.

4. SPECIFICITY TEST
Contains at least ONE: exact number, timestamp, named person/company, direct quote, physical detail?
If NO: specificity <= 4, score -= 1.

5. STRUCTURAL MIMICRY TEST (IF VIRAL REFERENCE EXISTS)
Count sentences in source opening. Count sentences in generated hook.
If mismatch: score -= 1, feedback must note "sentence count mismatch with source."

6. CREDENTIAL STACKING TEST
Does the hook open by establishing the author's credentials ("I build...", "I've spent X years...", "As a founder of...")?
Did the source post open this way?
If hook has credentials AND source did NOT: credential_stack_detected = true, score -= 2.

Apply these rules deterministically. Do not override. Do not explain.
