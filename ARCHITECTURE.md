## Overview
Digital DNA is a full-stack system designed to ingest, map, and utilize a founder's intellectual property inside a graph structure. It parses content, builds a dynamic knowledge graph (representing tone, beliefs, stories, etc.), and generates authentic social posts using LLMs.

## Components

### Backend Core (`src/`)
- **Purpose:** Handles the core business logic, including ingestion, graphing, vector search, LLM routing, generation agents, and humanization.
- **Location:** `src/`
- **Dependencies:** NetworkX, ChromaDB, LangChain, LangGraph, Spacy, PyYAML.
- **Dependents:** `webapp/server.py`, CLI commands (`src/cli.py`).

### FastAPI Server (`webapp/server.py`)
- **Purpose:** Exposes the core Python modules as REST endpoints (and SSE streams) for the frontend.
- **Location:** `webapp/server.py`
- **Dependencies:** `src/*`, FastAPI, Uvicorn, Subprocess.
- **Dependents:** React Frontend (`webapp-react/`).

### React Frontend (`webapp-react/`)
- **Purpose:** User interface for visualizing the knowledge graph, generating content, managing configuration, and triggering workflows.
- **Location:** `webapp-react/src/`
- **Dependencies:** React, Vite, TailwindCSS, Zustand, React Query, react-force-graph-2d.
- **Dependents:** External users.

## Data Flow
1. **Ingestion**: Raw content (like CSVs or podcast transcripts) are processed via Python scripts (`src/ingestion`).
2. **Graph Building**: Extracted entities are stored using NetworkX for relations, and ChromaDB for semantic search.
3. **API Interaction**: Frontend calls endpoints like `/api/graph/stats` or `/api/generate/topic`.
4. **Generation**: The backend leverages LangGraph workflows and configured LLM providers to construct posts based on graph context.

## Integration Points
| External Service | Type | Purpose |
|------------------|------|---------|
| Local FastAPI | API | Proxies all backend capabilities for the frontend |
| LLM Providers | API | Used for content extraction and generation (Anthropic, OpenAI, Ollama) |
| Local Filesystem | Storage | Graph structures saved as pickle objects; vector stores mapped to disk |

## Conventions
- **Naming:** Backend components use snake_case (Python standard), while Frontend components use PascalCase for components and camelCase for logic.
- **Structure:** Clear separation between standard Python core logic and Web endpoints (`webapp`). Frontend is totally isolated as a modern Vite/React SPA.
- **Testing:** Contains a `tests/` directory for standard Python testing configurations.
